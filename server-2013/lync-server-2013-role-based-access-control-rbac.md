# Rollenbasierte Zugriffskontrolle (RBAC) für Lync Server 2013

 

_**Letztes Änderungsdatum des Themas:** 2013-11-07_

Microsoft Lync Server 2013 umfasst rollenbasierte Zugriffssteuerungsgruppen (RBAC-Gruppen), mit denen Sie administrative Aufgaben delegieren und gleichzeitig hohe Sicherheitsstandards einhalten können. Diese Gruppen werden während der Gesamtstrukturvorbereitung erstellt. Ausführliche Informationen zur Gesamtstrukturvorbereitung finden Sie unter [Active Directory-Domänendienste für Lync Server 2013](lync-server-2013-active-directory-domain-services-for-lync-server.md). Einzelheiten zu den bestimmten Gruppen, die während der Gesamtstrukturvorbereitung erstellt werden, finden Sie in der Bereitstellungsdokumentation unter [Änderungen bei der Gesamtstrukturvorbereitung in Lync Server 2013](lync-server-2013-changes-made-by-forest-preparation.md).

Mit RBAC wird das administrative Recht gewährt, indem Benutzer vordefinierten administrativen Rollen zugewiesen werden, einschließlich der elf vordefinierten Rollen, die viele häufig verwendete administrative Aufgaben abdecken. Jede Rolle ist mit einer bestimmten Liste von Lync Server-Verwaltungsshell-cmdlets verbunden, die Benutzer in dieser Rolle ausführen dürfen. Mit der rollenbasierten Zugriffssteuerung können Sie dem Prinzip der geringsten Rechte folgen, bei dem Benutzer nur die administrativen Funktionen erhalten, die sie für ihre Arbeit benötigen. Ausführliche Informationen finden Sie in der Planungsdokumentation unter [Planen für die rollenbasierte Zugriffssteuerung in Lync Server 2013](lync-server-2013-planning-for-role-based-access-control.md).
